# ADB Tool 测试使用说明

本文档面向 Android App 测试人员，介绍 ADB Tool 的主要功能、典型使用流程和常见问题。测试同学可以通过该工具完成设备连接、投屏观察、日志查看、文件管理、应用安装卸载、截图录屏、截图标注、Intent / 深链接测试、剪贴板文本发送等日常测试操作。

## 1. 软件简介

ADB Tool 是一个图形化 Android 调试与测试工具，用于把常见 ADB 操作封装成桌面界面，减少测试过程中手动输入命令的成本。

它适合以下场景：

- 测试公司 Android App 的安装、启动、卸载和数据清理流程。
- 查看设备状态、系统信息、运行日志和后端执行日志。
- 抓取 Logcat 日志，辅助定位崩溃、卡顿、接口错误等问题。
- 浏览 Android 设备文件，上传、下载、删除测试文件。
- 使用 scrcpy 投屏观察或演示设备画面，并按设备保存投屏参数。
- 截图、录屏并保存测试证据。
- 对截图添加时间戳、步骤编号，并进行图片编辑标注。
- 测试 URL 跳转、App 深链接和自定义 Intent。
- 通过无线 ADB 连接设备，减少 USB 线缆依赖。

## 2. 使用前准备

### 2.1 手机侧准备

1. 打开 Android 手机的「开发者选项」。
2. 开启「USB 调试」。
3. 使用 USB 数据线连接电脑。
4. 手机弹出 USB 调试授权时，点击允许。
5. 如果需要使用无线 ADB，请确保电脑和手机在同一网络下。

### 2.2 电脑侧准备

1. 启动 ADB Tool 桌面应用。
2. 等待左侧设备列表自动刷新。
3. 如果设备显示为 `unauthorized`，请检查手机上是否确认了 USB 调试授权。
4. 如果设备未显示，可以尝试重新插拔 USB、重启 ADB 或检查数据线是否支持传输数据。

## 3. 主界面说明

软件左侧是设备列表和功能导航。每台已连接设备会在侧边栏展示设备图标，点击展开后可选择功能入口：

- 设备状态
- 投屏
- Logcat 日志
- 文件管理
- 应用管理
- 设备信息
- 剪贴板
- ADB 指令
- 设置

多台设备同时连接时，请先确认当前选中的设备，避免对错误设备执行安装、卸载、清数据、重启等操作。

支持使用 Cmd/Ctrl+K 快捷键快速打开命令面板，键盘导航切换功能页面，提升操作效率。

## 4. 设备状态

设备状态页面用于快速查看当前设备的基础运行情况。

可用于：

- 确认设备是否在线。
- 查看设备型号、品牌、系统版本等基础信息。
- 查看实时设备指标：CPU 使用率、内存占用、电池电量与温度、存储用量。
- 通过趋势图观察 CPU 和内存等指标随时间的变化。
- 判断当前测试设备是否连接正常。
- 在执行测试前确认目标设备是否正确。

建议在开始测试前先进入该页面确认设备状态，避免后续日志、安装、截图等操作选错设备。

## 5. Logcat 日志

Logcat 页面用于实时查看 Android 设备日志，是定位 App 崩溃、异常、接口错误和调试信息的核心页面。

### 5.1 主要能力

- 实时拉取 Logcat 日志。
- 支持暂停、继续、清空日志。
- 支持自动滚动到最新日志。
- 支持按日志级别过滤。
- 支持按 Tag、包名、PID、关键词过滤。
- 支持清理设备 Logcat 缓冲区。

### 5.2 常见使用方式

#### 查看 App 崩溃日志

1. 进入 Logcat 页面。
2. 输入被测 App 的包名或关键字进行过滤。
3. 复现 App 崩溃或异常问题。
4. 查找 `FATAL EXCEPTION`、`AndroidRuntime`、`Exception`、`Error` 等关键日志。
5. 将日志内容复制或截图保存到缺陷单。

#### 查看接口或业务日志

1. 根据项目约定输入 Tag 或关键词。
2. 执行业务操作。
3. 观察请求参数、响应结果、错误码或业务异常信息。

## 6. 文件管理

文件管理页面用于浏览 Android 设备文件系统，并进行常用文件操作。

### 6.1 快捷目录

顶部提供常用目录快捷入口：

- 根目录 `/`
- 用户存储目录 `/storage/emulated/0`
- Android 沙盒目录 `/sdcard/Android/data`
- 临时目录 `/data/local/tmp`

### 6.2 主要能力

- 浏览设备目录和文件。
- 上传文件到当前目录。
- 从设备下载文件到电脑。
- 删除文件或目录。
- 重命名文件或目录。
- 新建文件。
- 新建文件夹。
- 复制文件路径。
- 查看文件详情。
- 读取文本文件内容。
- 在当前设备上截图或录屏并保存到电脑。

### 6.3 典型测试场景

#### 查看 App 沙盒相关文件

1. 进入文件管理。
2. 点击 Android 沙盒目录快捷入口。
3. 找到被测 App 的包名目录。
4. 查看缓存、日志、下载文件或业务数据文件。

注意：不同 Android 版本和不同 ROM 对 `Android/data` 的访问限制可能不同。如果某些目录无法打开，通常是系统权限限制，并不一定是工具异常。

#### 导出测试文件

1. 找到目标文件。
2. 使用下载功能保存到电脑。
3. 将文件附加到缺陷单或测试报告。

#### 上传测试文件

1. 进入目标目录。
2. 使用上传功能选择电脑本地文件。
3. 上传后在 App 中触发相关读取逻辑。

## 7. 截图、录屏、预览和编辑

截图和录屏可在设备信息页或文件管理页中使用，用于保存测试证据。

### 7.1 截图流程

1. 点击截图按钮。
2. 工具从当前设备抓取屏幕截图。
3. 弹出水印选项。
4. 根据需要选择是否添加时间戳。
5. 根据需要选择是否添加步骤编号。
6. 如果选择步骤编号，需要手动输入编号，例如 `1`、`2`、`5`。
7. 选择快速保存，或进入图片编辑器。

### 7.2 截图水印

截图支持两类测试标记：

- 时间戳：记录截图产生的具体时间，方便和日志、测试步骤对应。
- 步骤编号：由测试人员手动输入，适合在缺陷复现步骤中标记第几步。

步骤编号不会自动累加，因为不同缺陷、不同测试流程可能需要从任意步骤开始标记。

### 7.3 图片编辑

进入图片编辑器后，可以对截图进行标注和编辑，例如：

- 裁剪。
- 旋转。
- 添加文字。
- 绘制线条或形状。
- 标记问题区域。
- 调整滤镜等。

编辑完成后再保存到电脑，适合用于缺陷截图、测试报告和问题说明。

### 7.4 快速保存

如果截图不需要编辑，可以选择快速保存。工具会直接将带有水印选项的截图保存到电脑。

### 7.5 录屏流程

1. 点击录屏开始。
2. 在手机上执行需要记录的操作。
3. 点击停止录屏。
4. 工具拉取录屏视频并打开预览。
5. 确认视频内容无误后保存到电脑。

录屏适合记录复杂复现路径、动画异常、页面跳转异常、闪退前操作过程等问题。

## 8. 应用管理

应用管理页面用于查看、安装、卸载和管理设备上的 App。

### 8.1 主要能力

- 查看已安装应用列表。
- 搜索应用包名。
- 区分系统应用和用户应用。
- 拖放安装 APK。
- 卸载应用。
- 强制停止 App。
- 清除 App 数据。

### 8.2 安装 APK

1. 进入应用管理页面。
2. 将 APK 文件拖入页面，或使用安装入口选择 APK。
3. 等待安装结果。
4. 如果安装失败，页面会展示失败原因。

常见安装失败原因包括：

- 签名不一致。
- 版本号低于已安装版本。
- 设备存储空间不足。
- APK 架构或系统版本不兼容。

### 8.3 卸载 App

1. 在应用列表中找到目标 App。
2. 点击更多操作。
3. 选择卸载。
4. 确认目标包名后执行。

卸载前请确认是否需要保留数据。部分测试场景可能需要先导出日志、缓存或数据库文件。

### 8.4 强制停止 App

强制停止会执行类似以下 ADB 操作：

```bash
adb shell am force-stop <package>
```

适合用于：

- 模拟 App 被系统或用户停止后的冷启动。
- 测试重新打开 App 后的状态恢复。
- 结束后台进程后重新复现问题。

### 8.5 清除 App 数据

清除数据会执行类似以下 ADB 操作：

```bash
adb shell pm clear <package>
```

适合用于：

- 恢复 App 到首次安装状态。
- 清除登录态、缓存、数据库和 SharedPreferences。
- 测试新用户流程。
- 复现首次启动、首次授权、首次引导等场景。

注意：清除数据是破坏性操作，执行前请确认包名，必要时先备份测试数据。

## 9. 设备信息

设备信息页面用于查看设备属性、系统信息和 `getprop` 相关内容。

可用于：

- 查看设备型号、品牌、系统版本。
- 查看系统属性。
- 辅助判断问题是否与 Android 版本、厂商 ROM 或设备型号有关。
- 在提交缺陷时补充设备环境信息。
- 执行截图和录屏操作。

建议在提交兼容性问题时，将设备型号、Android 版本、系统构建信息和问题截图一起提供。

## 10. 剪贴板

剪贴板页面用于从电脑向 Android 设备发送文本。已发送内容会保存到本地历史中，方便复用常用账号、链接、JSON 片段或测试文本。

### 10.1 使用方式

1. 进入剪贴板页面。
2. 如果提示未安装剪贴板助手，先安装辅助 APK。
3. 输入需要发送到手机剪贴板的文本。
4. 点击发送。
5. 在手机 App 的输入框中粘贴内容。

### 10.2 适用场景

- 输入长文本。
- 输入测试账号、验证码、链接、JSON、Token 等内容。
- 复用最近发送过的常用文本，或将高频文本标记为收藏。
- 避免在手机上手动输入复杂字符。

注意：不要通过剪贴板发送生产环境密码、真实用户隐私数据或其他敏感信息。

## 11. 投屏

投屏页面通过内置 scrcpy 启动设备画面镜像。支持同时投屏多台设备，每台设备独立窗口显示。工具内的页面主要用于选择设备、调整投屏参数、启动/停止投屏以及查看快捷键参考。

### 11.1 使用方式

1. 在左侧选择目标设备。
2. 进入投屏页面。
3. 根据需要调整投屏设置，例如最大尺寸、码率、帧率、编码、是否静音、是否窗口置顶、是否只读、是否保持唤醒。
4. 点击开始投屏。
5. 如果没有看到画面，请检查任务栏或 Dock 中是否已经打开 scrcpy 独立窗口。
6. 测试结束后点击结束投屏。

同时投屏多台设备时，可在投屏页面切换设备标签页分别管理每台设备的投屏状态和参数。

### 11.2 常用设置

- 视频源：可选择屏幕，也可在支持场景下选择摄像头。
- 视频：控制最大尺寸、码率、帧率、编码、裁剪和显示方式。
- 音频：控制是否采集声音、音频源和音频编码。
- 窗口：控制无标题栏、置顶、全屏和屏幕保护相关行为。
- 控制：选择键盘/鼠标模式，或开启只读模式避免误操作设备。
- 设备：保持唤醒、镜像时关闭手机屏幕、显示触摸点等。
- 录制：投屏同时保存录制文件，并可选择保存目录、格式和时长限制。也支持无窗口后台录制（不启动投屏窗口，仅录屏保存文件），适合录屏后无需实时观察的场景。

### 11.3 适用场景

- 复现问题时把手机画面显示到桌面，方便观察和演示。
- 测试会议或远程沟通中展示设备操作过程。
- 需要更大画面确认 UI 细节、动画、跳转和触摸反馈。
- 需要一边投屏一边录制复现过程。

### 11.4 注意事项

- 投屏依赖设备处于 ADB 在线状态，`unauthorized` 或 `offline` 时无法启动。
- 投屏窗口是 scrcpy 独立窗口，不嵌在工具页面里。
- 快捷键需要先点击 scrcpy 窗口，让该窗口获得焦点后才会生效。
- 开启“只读”后只能观察，不能通过电脑键鼠控制设备。
- 如果同时开启投屏录制，结束投屏时工具会优雅停止 scrcpy 进程，尽量保证录制文件完整写入。

## 12. ADB 指令

ADB 指令页面提供命令输入框和常用快捷指令，适合需要执行 ADB 命令但不想切换终端的测试场景。使用 Cmd/Ctrl+K 可快速打开命令面板，通过键盘搜索和执行任意快捷指令。

### 12.1 命令输入

可以输入 ADB 参数或完整 ADB 命令。例如：

```bash
shell getprop ro.product.model
```

也可以输入：

```bash
adb shell getprop ro.product.model
```

工具会自动使用当前选中的设备执行命令。

### 12.2 快捷指令分组

当前提供以下快捷操作组：

#### 设备信息

- 查看设备型号。
- 查看 Android 版本。
- 查看设备序列号。
- 查看电池状态。

#### 屏幕控制

- 查看屏幕分辨率。
- 查看屏幕密度。
- 唤醒屏幕。
- 模拟电源键。

#### 按键模拟

- Home 键。
- Back 键。
- 最近任务键。
- Menu 键。

#### 调试诊断

- 查看当前 Activity。
- 查看当前焦点窗口。
- 查看 CPU Top。
- 查看进程列表。

#### 存储和网络

- 查看存储空间。
- 查看网络地址。
- 查看路由信息。
- 查看连接状态。

#### 维护操作

- 清理 Logcat。
- 开启无线 ADB。
- 恢复 USB ADB。
- 重启设备。

维护类操作可能影响设备状态，执行前请确认目标设备。

## 13. Intent 和深链接测试

ADB 指令页面提供 Intent 测试相关快捷入口，适合测试 App 的 URL 跳转、Scheme、Universal Link、App Link 和自定义 Intent。

### 13.1 打开链接

用于测试普通 URL 是否能被系统正确打开。例如：

```text
https://example.com/path
```

工具会执行类似命令：

```bash
adb shell am start -a android.intent.action.VIEW -d https://example.com/path
```

### 13.2 深链接跳转

用于测试 App 自定义 Scheme 或 App Link。例如：

```text
myapp://product/detail?id=123
```

可用于验证：

- App 是否能被指定链接拉起。
- 路由参数是否解析正确。
- 未登录、已登录状态下跳转行为是否符合预期。
- 异常参数是否被安全处理。

### 13.3 自定义 Intent

自定义 Intent 表单支持填写：

- Action
- Data
- Category
- Extras
- Boolean Extras

适合用于测试：

- 指定 Action 的页面拉起。
- 带参数页面跳转。
- 推送落地页模拟。
- 外部 App 调起当前 App 的兼容性。
- 错误参数、缺失参数、边界参数处理。

### 13.4 深链接测试建议

建议测试同学至少覆盖：

1. 合法链接。
2. 缺少必填参数的链接。
3. 参数类型错误的链接。
4. 不存在资源 ID 的链接。
5. 未登录状态跳转。
6. 已登录状态跳转。
7. App 冷启动时跳转。
8. App 已在后台时跳转。
9. App 已在前台时跳转。

## 14. 无线 ADB

工具支持通过 ADB 快捷指令开启无线 ADB，也支持连接和断开无线设备。

### 14.1 常见流程

1. 手机通过 USB 连接电脑。
2. 在 ADB 指令页面执行开启无线 ADB。
3. 查看手机 IP 地址。
4. 使用无线连接能力连接设备。
5. 连接成功后可以拔掉 USB 线继续测试。

### 14.2 注意事项

- 电脑和手机需要在同一网络下。
- 网络不稳定时可能导致设备断开。
- 某些设备重启后需要重新开启无线 ADB。
- 测试敏感场景时，建议优先使用 USB 连接，稳定性更好。

## 15. 后端日志

后端日志页面用于查看本地 Go 后端执行 ADB 操作时产生的运行日志。

可用于：

- 判断工具后端是否正常启动。
- 查看 ADB 命令执行失败原因。
- 排查文件上传、下载、截图、录屏、安装 APK 等操作异常。
- 向开发同学反馈工具问题时提供辅助信息。

如果某个功能页面只提示操作失败，但原因不明确，可以打开后端日志页面查看更详细的错误信息。

## 16. 设置

设置页面提供统一的配置入口，可以调整应用行为和界面偏好。

### 16.1 主题切换

支持亮色和暗色两种主题。切换后即时生效，重启应用后保持上次选择。

### 16.2 语言切换

支持中文和英文界面。切换后即时生效，重启应用后保持上次选择。

### 16.3 录屏配置

可以设置录屏文件的默认存储路径和录制方式（ADB screenrecord 或 scrcpy 无窗口录制），适应不同测试环境的需求。

### 16.4 其他设置

通过设置页面可调整应用的其他全局行为，所有设置项持久化保存，重启后依然有效。

## 17. 测试会话与问题取证

Test Session 页面用于把一次测试过程中的截图、录屏、Logcat、问题标记和导出报告集中管理，适合缺陷复现、回归验证、兼容性测试和版本验收。

如果已经在测试配置中选择了当前被测 App，新建测试会话时会自动回显该配置里的测试流程/步骤。测试人员可以在新建会话弹窗中确认或临时调整流程，开始会话后这些流程会保存为本次会话的步骤快照。

### 17.1 功能区说明

页面顶部工具区提供常用操作：

- 新建会话：创建一次新的测试记录，填写测试名称、类型、包名、初始备注和测试流程/步骤；如果当前选中了测试配置，会自动回显配置中的流程/步骤。
- 录屏：记录完整复现过程，停止后保存为会话附件。
- 截图：抓取当前设备画面并保存到会话附件。
- Logcat：开始或停止持续日志采集，生成完整过程日志。
- 标记问题：填写结构化问题信息，并自动关联最近的截图、录屏和日志。
- 添加备注：记录普通测试备注。
- 导出：生成包含 `session.json`、`report.md`、截图、录屏和日志的问题包。
- 结束会话：结束当前测试会话并生成报告。

当窗口宽度较窄时，顶部工具区会自动换行显示。按钮显示不全时可以横向扩展窗口或使用换行后的按钮区域继续操作，不会影响测试会话功能。

### 17.2 推荐使用流程

1. 连接并选择目标设备。
2. 进入 Test Session 页面。
3. 点击新建会话，填写测试名称、测试类型和被测 App 包名；如果当前测试配置已有流程/步骤，确认回显内容是否正确。
4. 如需临时调整本次验证范围，可在新建会话弹窗中直接编辑测试流程/步骤。
5. 如需完整日志，先点击 Logcat 开始持续采集。
6. 需要记录动态过程时点击录屏。
7. 按右侧“当前测试内容”逐步执行测试，并对完成的步骤标记通过或失败原因。
8. 关键页面或异常现场点击截图。
9. 发现问题时点击标记问题，填写标题、类型、严重程度、复现步骤、预期结果和实际结果。
10. 停止录屏或停止 Logcat。
11. 点击导出，提交生成的问题包给开发同学。

### 17.3 测试流程/步骤

测试流程/步骤建议使用以下格式填写：

```text
登录流程：
- 打开 App
- 输入账号密码
- 点击登录

支付流程：
- 进入收银台
- 确认支付
```

每个以冒号结尾的行会作为流程名称，下面的 `-`、数字编号或普通文本行会作为步骤。新建测试配置和编辑测试配置中保存的流程，可在新建测试会话时自动带出；会话开始后会保存为会话快照，不会因为后续修改测试配置而影响历史会话。

### 17.4 标记问题

标记问题建议至少填写：

- 问题标题。
- 问题类型，例如崩溃、ANR、性能问题、UI 异常、接口异常、功能异常、兼容性或其他。
- 严重程度，例如阻塞、严重、一般或轻微。
- 复现步骤。
- 预期结果。
- 实际结果。
- 补充备注。

标记问题后，工具会在会话中生成独立的问题记录，并在报告的“问题摘要”中优先展示。

### 17.5 最近 1000 行 Logcat 快照

点击标记问题时，工具会尝试额外获取当前设备最近 1000 行 Logcat，并保存为独立文件：

```text
issue_logs/ISSUE-001_last_1000_yyyyMMdd_HHmmss.log
```

该文件用于快速定位问题现场，不会停止或覆盖正在进行的持续 Logcat 采集，也不会清空设备 Logcat 缓冲区。

如果获取最近日志失败，标记问题本身仍会继续保存，只是该问题不会附带最近 1000 行日志快照。

### 17.6 导出问题包

导出后通常包含：

- `session.json`：测试会话的结构化数据。
- `report.md`：面向开发的问题摘要和测试过程报告。
- `screenshots/`：会话截图。
- `videos/`：会话录屏。
- `logs/`：持续采集或手动保存的 Logcat 文件。
- `issue_logs/`：标记问题时自动保存的最近 1000 行 Logcat 快照。

建议提交缺陷时优先附带 `report.md` 和对应问题的 `issue_logs` 文件，必要时再附带完整录屏和完整 Logcat。

### 17.7 运行时长

会话开始后，顶部状态区域会实时显示当前已运行时长（HH:MM:SS 格式）。该时长直接根据会话启动时间计算，即使切到其他页面再回来也不会重置。

### 17.8 删除单个附件

侧边栏附件区域列出每个附件的名称和大小。点击附件右侧的 × 按钮可以单独删除该附件。删除前会弹出确认对话框，确认后附件文件被永久删除，同时从会话记录中移除。

**注意**：只有运行中的会话可以删除附件。历史/已结束的会话为只读模式，不能删除附件。

### 17.9 历史会话浏览

在暂无会话的空白页，或顶部工具区的历史记录按钮，可以打开历史会话对话框：

- 按时间倒序列出所有历史会话（最新优先）
- 每个会话显示：名称、测试类型、设备、时间、问题数量
- **重新打开**：以只读模式加载历史会话，可查看完整时间线和附件，但不能修改
- **删除会话**：永久删除会话目录及所有附件，删除前有确认对话框
- 对话框中可连续删除多个会话，列表实时更新

### 17.10 会话存储位置

所有会话数据存储在用户主目录下的 `ADBToolData/sessions/` 目录中：

```text
~/ADBToolData/sessions/
├── 20260615_120000_缺陷复现测试/
│   ├── session.json
│   ├── report.md
│   ├── screenshots/
│   ├── videos/
│   ├── logs/
│   └── issue_logs/
└── 20260614_090000_冒烟测试/
    └── ...
```

## 18. 推荐测试流程

### 18.1 新版本 App 基础验证流程

1. 连接测试设备。
2. 在设备状态页确认设备在线。
3. 进入应用管理，安装待测 APK。
4. 如需干净环境，先清除 App 数据。
5. 启动 App 并执行基础冒烟流程。
6. 打开 Logcat，按包名过滤日志。
7. 发现问题时截图，并添加时间戳和步骤编号。
8. 如问题需要动态过程说明，使用录屏保存复现过程。
9. 导出必要日志、截图、录屏和设备信息。
10. 提交缺陷单。

### 18.2 缺陷复现取证流程

1. 在应用管理中强制停止 App。
2. 根据需要清除 App 数据。
3. 打开 Logcat 并清空旧日志。
4. 开始执行复现步骤。
5. 每个关键步骤截图，并手动输入对应步骤编号。
6. 问题出现时保存截图或录屏。
7. 从 Logcat 中筛选异常日志。
8. 将截图、录屏、日志和设备信息一起提交。

### 18.3 深链接测试流程

1. 准备测试链接列表。
2. 连接设备并确认目标 App 已安装。
3. 根据测试场景选择未登录或已登录状态。
4. 在 ADB 指令页面使用深链接跳转。
5. 验证页面是否正确打开。
6. 验证参数是否正确展示或处理。
7. 对异常链接记录截图和日志。
8. 对冷启动、后台、前台三种状态分别验证。

### 18.4 App 崩溃排查流程

1. 打开 Logcat 页面。
2. 清空旧日志。
3. 按包名或关键 Tag 过滤。
4. 复现崩溃。
5. 查找 `FATAL EXCEPTION`、`AndroidRuntime` 或业务异常关键字。
6. 保存崩溃日志。
7. 截图或录屏记录崩溃前操作路径。
8. 附带设备信息提交问题。

## 19. 常见问题

### 19.1 为什么根目录下有 `/sdcard`，但推荐使用 `/storage/emulated/0`？

很多 Android 设备中，`/sdcard` 并不是真实目录，而是指向用户存储目录的软链接，通常指向：

```text
/storage/emulated/0
```

不同 Android 版本和厂商 ROM 对软链接展示、权限和路径解析的处理可能不同。为了减少兼容性问题，查看用户存储目录时优先使用 `/storage/emulated/0`。

工具中已经对部分绝对路径软链接做了真实路径解析，但测试时仍建议优先使用用户存储快捷入口。

### 19.2 为什么 `/sdcard/Android/data` 有时可以进，有时不能进？

`Android/data` 属于 App 外部沙盒目录。Android 11 之后，系统对该目录访问限制更严格，不同厂商 ROM 的限制也不同。

可能出现：

- 可以看到目录，但无法进入某些 App 包名目录。
- 可以进入目录，但无法读取部分文件。
- USB ADB 可以访问，但普通文件管理器不能访问。
- 不同设备表现不同。

这通常是系统权限策略导致，不一定是工具异常。

### 19.3 设备显示 unauthorized 怎么办？

请检查手机是否弹出 USB 调试授权弹窗，并点击允许。如果没有弹窗，可以尝试：

1. 重新插拔 USB。
2. 切换 USB 连接模式。
3. 在开发者选项中撤销 USB 调试授权后重新连接。
4. 更换数据线或 USB 接口。

### 19.4 截图步骤编号为什么需要手动输入？

测试过程中不同缺陷、不同用例可能从不同步骤开始记录。如果自动递增，容易把新缺陷的步骤编号接在旧缺陷后面，导致缺陷描述混乱。因此工具采用手动输入方式，由测试人员按当前用例或缺陷步骤填写编号。

### 19.5 清除数据后 App 登录态没了正常吗？

正常。清除数据会删除 App 的本地数据，包括登录态、缓存、数据库、SharedPreferences 等。执行前请确认该操作符合当前测试目的。

### 19.6 录屏保存前为什么要预览？

预览可以确认录屏是否成功、内容是否完整、是否录到关键复现步骤。确认无误后再保存，可以减少无效录屏文件。

### 19.7 执行 ADB 指令失败怎么办？

可以按以下顺序排查：

1. 确认设备在线。
2. 确认当前选中的设备正确。
3. 检查命令参数是否正确。
4. 查看命令输出区域的错误信息。
5. 打开后端日志查看更详细的失败原因。

## 20. 提交缺陷时建议附带的信息

为了提高问题定位效率，建议缺陷单至少包含：

- App 版本号。
- 设备型号。
- Android 系统版本。
- 测试账号或测试环境。
- 复现步骤。
- 期望结果。
- 实际结果。
- 带步骤编号和时间戳的截图。
- 必要时附带录屏。
- 崩溃或异常相关 Logcat 日志。
- 如果与文件有关，附带导出的文件或文件路径。
- 如果与深链接有关，附带完整链接和登录状态。

## 21. 操作安全提醒

以下操作会影响设备或 App 状态，执行前请确认目标设备和包名：

- 卸载 App。
- 清除 App 数据。
- 强制停止 App。
- 删除设备文件。
- 重启设备。
- 开启或切换无线 ADB。
- 执行自定义 ADB 命令。

建议测试过程中保持一台设备只执行一个明确测试任务，避免多设备、多页面同时操作导致误操作。
